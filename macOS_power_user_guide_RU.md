## Как пользоваться этим руководством

Не пытайся прочитать и применить всё за вечер. У документа три режима:

1. **Перед началом работы:** выполни чек-листы «первые 60 минут», «первый вечер» и «первая неделя».
2. **Обучение:** последовательно пройди главы о модели macOS, клавишах, Finder, окнах и терминале.
3. **Справочник:** возвращайся к разделам о Docker, Java, безопасности, бэкапах и диагностике, когда появится реальная задача.

После освоения гайда ты должен уметь:

- работать почти без мыши, быстро находить приложения, файлы, команды и настройки;
- понимать, где лежат файлы и настройки, чем приложение отличается от окна и почему красная кнопка не всегда завершает программу;
- уверенно пользоваться Finder, Spotlight, Spaces, Mission Control, Quick Look и системными сочетаниями клавиш;
- безопасно ставить, обновлять и удалять приложения;
- понимать основы Unix: shell, пути, права, процессы, переменные окружения и `PATH`;
- поддерживать воспроизводимое dev-окружение для Java/Kotlin, Git, IntelliJ IDEA и Docker;
- распознавать проблемы ARM64/x86_64 и не маскировать их бездумной установкой Rosetta;
- иметь проверяемый бэкап, включённое шифрование и план восстановления;
- диагностировать зависшее приложение, нехватку диска, высокую нагрузку, проблемы сети и контейнеров.

## 1. План запуска нового MacBook

### Первые 60 минут

- [ ] Осмотри корпус и экран, проверь зарядку, порты, динамики, камеру и клавиатуру.
- [ ] Подключись к доверенной сети Wi‑Fi.
- [ ] Войди в свой Apple Account. Включи двухфакторную аутентификацию, если она ещё не включена.
- [ ] Создай **сильный уникальный пароль** учётной записи Mac. Touch ID — удобное дополнение, не замена паролю.
- [ ] Добавь минимум два отпечатка Touch ID: удобные пальцы разных рук.
- [ ] Обнови macOS: **Apple menu → System Settings → General → Software Update**.
- [ ] Включи **Find My Mac** и проверь, что Mac виден в приложении Find My.
- [ ] Открой **Privacy & Security → FileVault**, включи FileVault и надёжно сохрани способ восстановления. Потеря и пароля, и recovery key может означать потерю данных.
- [ ] Включи автоматические системные обновления и установку security responses.
- [ ] В **Lock Screen** потребуй пароль сразу или почти сразу после сна/скринсейвера.
- [ ] Настрой раскладки и клавишу переключения языка. Удали раскладки, которыми не пользуешься.
- [ ] Открой настройки Trackpad и включи жесты, которые описаны ниже. Не меняй сразу всё: дай стандартному поведению несколько дней.
- [ ] В Finder включи строку пути и строку состояния: **View → Show Path Bar**, **View → Show Status Bar**.
- [ ] В Finder → Settings → Advanced включи показ расширений файлов.
- [ ] Подключи внешний SSD и настрой Time Machine. Если диска пока нет — добавь его в ближайшие покупки, но не считай iCloud полноценным бэкапом.

### Первый вечер

- [ ] Перенеси только нужные документы. Не тащи на новый Mac старый мусор и случайные системные папки Windows.
- [ ] Установи рабочие приложения из официальных источников.
- [ ] Установи Xcode Command Line Tools: `xcode-select --install`.
- [ ] Установи Homebrew в стандартный путь `/opt/homebrew`.
- [ ] Настрой Git, SSH-ключ и подпись коммитов согласно правилам компании.
- [ ] Установи JetBrains Toolbox и нативную Apple Silicon-версию IntelliJ IDEA.
- [ ] Установи JDK через SDKMAN! либо другой один выбранный менеджер версий. Не смешивай без причины три разных менеджера JDK.
- [ ] Установи Docker Desktop for Apple silicon; проверь архитектуру тестового контейнера.
- [ ] Проверь первую резервную копию Time Machine и попробуй открыть интерфейс восстановления файлов.

### Первая неделя

- [ ] Выучи 20 основных сочетаний клавиш из раздела 4.
- [ ] Настрой 3–5 Spaces: например Communication, IDE, Browser/Docs, Terminal/Containers.
- [ ] Создай `Brewfile` или другой декларативный список установленного инструментария.
- [ ] Удали ненужные Login Items и проверь фоновые разрешения приложений.
- [ ] Настрой Focus для глубокой работы и отдельный Focus для созвонов.
- [ ] Проверь, что репозитории собираются **нативно на ARM64**.
- [ ] Проверь сценарий `git clone → JDK → build → tests → Docker Compose` с нуля на одном проекте.
- [ ] Оставь на SSD запас свободного места; при 1 ТБ разумная рабочая цель — не опускаться надолго ниже 15–20% свободного пространства.

## 2. Модель macOS: что нужно переучить после Windows

### Приложение, окно и документ — разные сущности

В Windows закрытие последнего окна часто завершает программу. В macOS приложение может остаться запущенным без окон.

| Действие | Сочетание | Что происходит |
|---|---:|---|
| Закрыть текущее окно/вкладку | `⌘W` | Приложение обычно остаётся запущенным |
| Завершить приложение | `⌘Q` | Закрывается всё приложение |
| Скрыть приложение | `⌘H` | Окна исчезают, процесс остаётся |
| Свернуть окно в Dock | `⌘M` | Сворачивается конкретное окно |
| Принудительно завершить | `⌥⌘Esc` | Только для зависших приложений |

Красная кнопка закрывает окно, жёлтая сворачивает, зелёная обычно разворачивает/плиточно размещает окно. `⌘Q` — аналог осознанного Exit. Точка под значком в Dock означает, что приложение запущено.

### Строка меню принадлежит активному приложению

Верхняя строка меню едина для всего экрана. Слева после логотипа Apple отображается имя **активного приложения**, а команды меню меняются вместе с ним. Если не понимаешь, почему сочетание не работает, сначала посмотри, какое приложение активно.

Полезная привычка: изучай пункты меню нового приложения. Справа от команды часто показано её сочетание клавиш.

### Dock — не меню «Пуск»

Dock — панель избранных и запущенных приложений, свёрнутых окон и корзины. Не превращай его в склад из 60 значков. Оставь часто используемое, остальное запускай через Spotlight.

- Запуск: `⌘Space`, несколько букв названия, `Enter`.
- Убрать значок из Dock: перетащить наружу или **Options → Remove from Dock**.
- Это не удаляет приложение.
- Правый клик по значку показывает окна и дополнительные действия.

### Установка `.app` — это часто простое копирование

Приложение macOS обычно является пакетом `.app`, который Finder показывает как один объект. Типичный `.dmg` — образ диска:

1. открыть `.dmg`;
2. перетащить приложение в **Applications**;
3. дождаться копирования;
4. извлечь смонтированный образ через `⏏` в Finder;
5. удалить скачанный `.dmg`.

Файл `.pkg` — установщик, который может размещать компоненты в нескольких местах. Его используй только от доверенного разработчика.

### Сон — нормальный режим

MacBook обычно не нужно выключать каждый вечер. Закрой крышку — система уснёт. Перезагрузка полезна после обновлений, при системных странностях или для диагностики. Перед долгим хранением устройство лучше выключить.

## 3. Символы клавиш и соответствия Windows

| Символ | Название | Роль |
|---:|---|---|
| `⌘` | Command | Главный модификатор приложений; чаще всего заменяет Windows `Ctrl` |
| `⌥` | Option / Alt | Альтернативное действие, специальные символы, навигация по словам |
| `⌃` | Control | Системные сочетания и терминал; не полный аналог Windows `Ctrl` |
| `⇧` | Shift | Регистр, обратное направление, расширение выделения |
| `fn` / `🌐` | Function / Globe | F-клавиши, emoji, раскладки и дополнительные действия |
| `⌫` | Delete | Удаляет символ слева, как Backspace |
| `fn⌫` | Forward Delete | Удаляет символ справа, как Delete на PC |
| `↩` | Return | Enter |
| `⎋` | Escape | Escape |

### Быстрая карта перехода

| Windows | macOS | Комментарий |
|---|---|---|
| `Ctrl+C/V/X/Z` | `⌘C/V/X/Z` | Копирование, вставка, вырезание, отмена |
| `Alt+Tab` | `⌘Tab` | Переключение приложений, а не отдельных окон |
| `Alt+F4` | `⌘Q` | Завершение приложения |
| `Ctrl+W` | `⌘W` | Закрытие вкладки/окна |
| `Win` и поиск | `⌘Space` | Spotlight |
| `Win+L` | `⌃⌘Q` | Блокировка экрана |
| `Ctrl+Shift+Esc` | Spotlight → Activity Monitor | Диспетчер задач |
| `F2` для переименования | `Return` в Finder | Переименовать выбранный файл |
| `Delete` файла | `⌘Delete` | Переместить в корзину |
| `Shift+Delete` | Обычно не нужен | Удали в корзину, затем очисти её осознанно |
| `Home/End` | `⌘← / ⌘→` | Начало/конец строки во многих редакторах |
| `Ctrl+←/→` | `⌥← / ⌥→` | По словам |
| `Ctrl+A` в поле | `⌘A` | Выделить всё |

В терминале `Ctrl+C` по-прежнему означает прервать процесс, а копирование — `⌘C`. Контекст решает всё.

## 5. Трекпад: главный инструмент MacBook

Force Touch-трекпад не обязан физически двигаться: ощущение клика создаёт Taptic Engine. Сила клика и жесты настраиваются в **System Settings → Trackpad**.

Рекомендуемый базовый набор:

- **Tap to click** — включить, если нравится лёгкое касание вместо нажатия;
- **Secondary click** — клик двумя пальцами;
- прокрутка двумя пальцами;
- масштабирование щипком;
- Mission Control — свайп вверх тремя или четырьмя пальцами;
- App Exposé — свайп вниз тремя или четырьмя пальцами;
- переход между полноэкранными приложениями/Spaces — свайп влево/вправо;
- показ рабочего стола — развести большой палец и три пальца;
- Look up & data detectors — полезно, но отключи, если вызывает случайные срабатывания.

**Natural scrolling** сначала кажется перевёрнутой после Windows: контент движется как лист под пальцами. Попробуй пожить с ним 3–5 дней. Если мозг всё ещё протестует — отключи; это не экзамен на «правильного маковода».

При работе с внешней мышью можно настроить её отдельно. Не ставь низкоуровневые драйверы и перехватчики ввода без необходимости: после обновлений macOS именно они часто становятся источником странностей.

## 6. Окна, Mission Control, Spaces и рабочие режимы

### Базовая стратегия

Для разработчика хорошо работает сочетание **Spaces + Mission Control**, а Stage Manager стоит считать опциональным режимом.

Пример Spaces:

1. Communication: Slack/Telegram/Mail/Calendar.
2. IDE: IntelliJ IDEA.
3. Browser & Docs.
4. Terminal & Docker.
5. Full-screen созвон или отдельная задача.

Открыть Mission Control — жест вверх или `⌃↑`. Вверху добавляются рабочие столы. Окно можно перетащить на нужный Space.

### Важные нюансы

- `⌘Tab` переключает **приложения**, а `⌘` + клавиша `` ` `` над Tab — окна внутри активного приложения.
- Полноэкранное окно создаёт отдельный Space. Это удобно, но чрезмерный full screen замедляет переходы.
- Зелёная кнопка и долгое наведение на неё предлагают полноэкранный/плиточный режим.
- В Dock → правый клик по приложению → Options можно закрепить приложение за конкретным Desktop.
- Если Spaces сами меняются местами, отключи **Automatically rearrange Spaces based on most recent use** в настройках Mission Control.
- Несколько мониторов: опция **Displays have separate Spaces** меняет модель работы и требует выхода из аккаунта для применения.

### Stage Manager

Stage Manager группирует окна и показывает недавние наборы сбоку. Он полезен на небольшом экране при работе с одной группой окон, но не обязателен. Сначала освой обычные окна и Spaces; потом реши, помогает ли он твоему стилю.

### Практика на 10 минут

1. Создай три Desktop.
2. Размести IDEA, браузер и терминал на разных Spaces.
3. Переключайся жестом и `⌃←/→`.
4. Через Mission Control перетащи окно на другой Space.
5. Научись делать это, не глядя на Dock.

## 7. Finder и файловая система

### Домашняя папка

Твоя домашняя директория — `/Users/<имя>` или сокращённо `~`.

Обычно внутри:

```text
~/Desktop       рабочий стол
~/Documents     документы
~/Downloads     загрузки
~/Pictures      изображения и библиотеки фото
~/Movies        видео
~/Music         музыка
~/Library       настройки, кэши и данные приложений
```

Системная `/Library` и пользовательская `~/Library` — разные места. Не удаляй из них файлы «для очистки», если не понимаешь назначение конкретного объекта.

### Корень системы

```text
/Applications   приложения для всех пользователей
/System         защищённые компоненты macOS
/Library        общесистемные настройки и данные
/Users          домашние папки пользователей
/Volumes        смонтированные диски и образы
/opt/homebrew   Homebrew на Apple Silicon
/tmp            временные данные
```

macOS использует APFS, разделяет системный и пользовательский тома и защищает критические части системы. Попытка «взять полный контроль» отключением SIP почти никогда не нужна для нормальной разработки.

### Настройки Finder, которые стоит включить

Finder → Settings:

- **General:** новые окна открывать в домашней папке, а не в Recents, если так понятнее;
- **Sidebar:** Home, Applications, Downloads, Documents, нужные iCloud-разделы и внешние диски;
- **Advanced:** показывать все расширения файлов;
- включить предупреждение перед изменением расширения;
- выбрать, очищать ли Trash автоматически — только если ты понимаешь последствия.

View:

- **Show Path Bar** — виден путь и можно переходить по его сегментам;
- **Show Status Bar** — количество объектов и свободное место;
- `⌘2` — список, `⌘3` — колонки, `⌘4` — галерея.

### Quick Look — привычка power user

Выдели файл и нажми `Space`: предпросмотр PDF, изображения, видео, документа, архива или исходника без запуска тяжёлого приложения. Стрелками можно переходить по соседним файлам.

### Поиск

- **Spotlight** (`⌘Space`) — быстрый глобальный поиск и запуск.
- **Finder Search** (`⌘F`) — сложные условия: тип, дата, имя, содержимое.
- **Smart Folder** — сохранённый поисковый запрос; файлы физически не перемещаются.
- Tags — метаданные, а не папки. Используй умеренно: например `Active`, `Waiting`, `Archive`.

### Операции, которые удивляют после Windows

- В Finder нет привычного `⌘X` для вырезания файла: нажми `⌘C`, затем `⌥⌘V` в месте назначения.
- `Return` переименовывает; открыть — `⌘O` или двойной клик.
- Файл с точкой в начале имени скрыт. Показать скрытые — `⇧⌘.`.
- Внешний диск нужно извлекать перед отключением, особенно если шла запись.
- NTFS обычно читается, но штатная запись ограничена. Для совместного диска Mac/Windows чаще подходит exFAT; для Mac-only — APFS. exFAT не даёт всех возможностей APFS и не заменяет бэкап.

### iCloud Drive — синхронизация, не единственная копия

iCloud удобен для документов и работы между устройствами. Но удаление/повреждение может синхронизироваться. Для важных данных нужна независимая резервная копия. Осторожно включай **Desktop & Documents Folders**: сначала пойми, куда переместятся существующие данные и сколько места доступно в iCloud.

## 8. Установка, обновление и удаление программ

### Приоритет источников

1. Mac App Store — удобно, sandboxed, автоматические обновления, но иногда версия функционально ограничена.
2. Официальный сайт разработчика — часто лучший вариант для dev-инструментов.
3. Homebrew — удобен для CLI и воспроизводимой установки GUI casks.
4. Неизвестный mirror, «кряк», случайный скрипт — не использовать.

Перед установкой проверь:

- это официальный домен;
- есть версия **Apple silicon / arm64** или Universal;
- приложение получает обновления;
- запрашиваемые разрешения соответствуют функции;
- лицензия подходит для рабочей/корпоративной среды.

### Gatekeeper и предупреждения

Не отключай Gatekeeper глобально. Если доверенное приложение заблокировано, открой **Privacy & Security** и разреши запуск конкретного приложения после проверки источника. `Control`-клик → Open тоже даёт контролируемый путь. Никогда не выполняй найденную в комментариях команду, которая массово снимает quarantine, если не понимаешь последствия.

### Удаление

Для простого `.app` часто достаточно перетащить его из Applications в Trash. Но приложения с helper-процессами, драйверами, VPN, антивирусами или системными расширениями лучше удалять официальным uninstaller. Homebrew cask: `brew uninstall --cask <name>`.

Остатки в `~/Library` обычно не опасны. Не стоит удалять всё по совпадению имени: можно снести общие данные или настройки другого продукта.

### Homebrew

Homebrew — пакетный менеджер. На Apple Silicon стандартный prefix — `/opt/homebrew`; это важно для готовых ARM64-пакетов.

Сначала:

```bash
xcode-select --install
```

Устанавливай Homebrew только командой с официальной страницы <https://brew.sh/>. После установки выполни предложенную установщиком команду `brew shellenv`, чтобы добавить Homebrew в `PATH`.

Основы:

```bash
brew update                 # обновить сведения о пакетах
brew upgrade                # обновить установленные пакеты
brew search ripgrep         # найти пакет
brew info ripgrep           # информация до установки
brew install ripgrep        # CLI-пакет
brew install --cask iterm2  # GUI-приложение, пример
brew list                   # установленное
brew outdated               # доступные обновления
brew uninstall ripgrep      # удалить
brew doctor                 # диагностика Homebrew
brew cleanup                # удалить старые версии и кэш по правилам brew
```

Не запускай `brew` через `sudo`. Не меняй владельца системных каталогов ради исправления Homebrew.

### Brewfile: воспроизводимый Mac

```bash
brew bundle dump --file=~/Brewfile --force
brew bundle check --file=~/Brewfile
brew bundle install --file=~/Brewfile
```

Просмотри файл перед сохранением в dotfiles: там могут быть рабочие приложения, которые ты не хочешь публиковать.

Пример минимального `Brewfile`:

```ruby
brew "git"
brew "jq"
brew "ripgrep"
brew "fd"
brew "fzf"
brew "htop"
brew "shellcheck"
cask "jetbrains-toolbox"
cask "docker"
cask "visual-studio-code"
```

Официальная документация установки: <https://docs.brew.sh/Installation>.

## 9. Безопасность без паранойи

### Обязательный минимум

- FileVault включён.
- Сильный пароль и Touch ID.
- Find My Mac включён.
- Автообновления macOS и приложений включены.
- Экран блокируется при уходе: `⌃⌘Q`.
- Отдельный password manager; уникальные пароли; 2FA, предпочтительно passkeys или аппаратный ключ для критичных аккаунтов.
- Резервные копии зашифрованы и проверены восстановлением.
- Установка только из доверенных источников.
- В повседневной работе не используется `root`.

### Разрешения Privacy & Security

macOS отдельно контролирует:

- Location, Contacts, Calendars, Photos;
- Camera и Microphone;
- Screen & System Audio Recording;
- Accessibility — очень мощное разрешение, позволяющее управлять интерфейсом;
- Full Disk Access — доступ к защищённым данным;
- Input Monitoring;
- Automation — управление другими приложениями;
- System Extensions и Network Filters.

Правило: выдавай разрешение только в момент, когда понятна функция. Раз в месяц просмотри список. Если приложение для обоев просит Full Disk Access или менеджер буфера требует доступ, который не объясняет, остановись и проверь.

### FileVault и ключ восстановления

На Apple Silicon внутреннее хранилище шифруется аппаратно, а FileVault добавляет защиту, связанную с учётными данными пользователя. Храни recovery key отдельно от ноутбука — например, в password manager и/или офлайн в безопасном месте. Не фотографируй ключ и не оставляй единственную копию в незашифрованной заметке на том же Mac.

Официальное руководство: <https://support.apple.com/guide/mac-help/protect-data-on-your-mac-with-filevault-mh11785/mac>.

### Нужен ли антивирус

Для аккуратного домашнего разработчика встроенные механизмы macOS, обновления и нормальная гигиена обычно важнее случайного «оптимизатора с антивирусом». Корпоративная политика может требовать EDR/антивирус — тогда используй одобренный компанией агент. Антивирус не делает безопасным запуск неизвестных скриптов, пиратского ПО и макросов.

### Чего не делать

- Не отключать SIP, Gatekeeper и FileVault ради сомнительного приложения.
- Не вставлять в Terminal команды вида `curl ... | sudo sh`, если не проверен официальный источник и содержимое.
- Не давать Terminal/IDE Full Disk Access «на всякий случай».
- Не хранить секреты в `.zshrc`, репозитории, Dockerfile или shell history.
- Не устанавливать «RAM cleaner». Не очищать системные кэши по расписанию.
- Не использовать один пароль для Mac, Apple Account, почты и GitHub.

### Секреты разработчика

- Используй password manager, macOS Keychain или корпоративное secret-хранилище.
- Для GitHub/GitLab — SSH-ключи или короткоживущие токены, passphrase на приватном ключе.
- Добавляй `.env` в `.gitignore`, но помни: `.gitignore` не удаляет уже закоммиченный секрет.
- Для облаков применяй отдельные профили и минимальные права.
- Перед демонстрацией экрана включай Focus, скрывай уведомления и проверяй вкладки/терминал.

## 10. Бэкап и восстановление

### Правило 3–2–1

Для действительно важных данных:

- 3 копии данных;
- 2 разных типа носителя/хранилища;
- 1 копия вне устройства/локации.

Практичный вариант:

1. рабочие данные на Mac;
2. автоматический Time Machine на зашифрованный внешний SSD/NAS;
3. критичные документы в доверенном облачном сервисе с историей версий либо отдельный off-site backup.

Git remote — копия исходного кода, но не всего окружения и не обязательно полноценный бэкап. iCloud/Dropbox — в первую очередь синхронизация.

### Time Machine

Путь: **System Settings → General → Time Machine → Add Backup Disk**.

Рекомендации:

- диск желательно больше реально занятого пространства; 2 ТБ для Mac на 1 ТБ обычно комфортнее, чем диск ровно на 1 ТБ;
- включи шифрование резервной копии;
- не исключай из бэкапа важные рабочие папки ради пары гигабайт;
- большие воспроизводимые кэши Docker/Gradle можно исключить, если точно понимаешь цену восстановления;
- периодически проверяй дату последнего успешного бэкапа;
- раз в квартал восстанови тестовый файл в отдельную папку и сравни его.

Официальная настройка: <https://support.apple.com/en-us/104984>.

### Что нужно сохранить отдельно

- recovery key FileVault;
- коды восстановления 2FA;
- доступ к password manager;
- SSH/GPG-ключи согласно политике компании;
- список приложений (`Brewfile`);
- dotfiles без секретов;
- инструкции по настройке dev-окружения;
- экспорт важных локальных баз, если они не воспроизводимы;
- лицензии и конфигурации критичного ПО.

### Проверь план восстановления

Ответь на вопросы:

- Если Mac украдут сегодня, какие данные исчезнут навсегда?
- Могу ли я войти в Apple Account без этого Mac и без SIM-карты?
- Где recovery key?
- Сколько времени займёт получение рабочего окружения на новом устройстве?
- Какие данные лежат только в Docker volume или локальной БД?

Если ответ неизвестен — бэкап ещё не настроен.

## 11. Аккумулятор, температура, диск и уход

### Аккумулятор

- Оставь включённой Optimized Battery Charging.
- Если твоя версия macOS предлагает **Charge Limit**, используй 80–90% при постоянной работе от сети и «Charge to Full Now» перед поездкой.
- Не устраивай регулярные циклы 0→100 «для калибровки».
- Главный враг батареи — жара. Не рендери/не обучай модель под одеялом и не оставляй ноутбук в горячей машине.
- Можно работать от сети; система управляет зарядкой. Не нужно вручную дёргать кабель каждые полчаса.
- Для хранения на недели: примерно половина заряда, выключенное устройство, прохладное сухое место.

Официальная информация о зарядке: <https://support.apple.com/en-us/102338>.

### Свободное место

SSD нужен системе для swap, временных файлов, обновлений, локальных snapshots и сборок. При 1 ТБ не считай последние гигабайты рабочими.

Проверяй:

- **System Settings → General → Storage**;
- крупные Downloads и старые `.dmg`;
- Docker images, volumes и build cache;
- Gradle/Maven caches;
- симуляторы Xcode, если появятся;
- локальные AI-модели;
- старые IDE и JDK;
- корзину.

Команды для обзора:

```bash
df -h /                         # свободное место системного тома
du -sh ~/Downloads/* 2>/dev/null
docker system df               # место Docker, ничего не удаляет
```

Не запускай слепо `docker system prune -a --volumes`: команда может удалить образы, кэши и volumes с нужными локальными данными.

### Экран и корпус

- Отключи питание, используй мягкую безворсовую ткань.
- Не распыляй жидкость прямо на экран.
- Не клади накладки на клавиатуру, камеру или ладонную область перед закрытием крышки: зазор очень мал.
- Не закрывай крышку с предметом на клавиатуре.
- Используй качественный USB‑C/Thunderbolt dock и кабели, соответствующие требуемой мощности/скорости.

## 12. Terminal и Unix для новичка

macOS — Unix-система. Стандартная интерактивная оболочка — `zsh`. Terminal запускает shell; shell читает команды и запускает программы.

### Базовые понятия

- **Текущая директория** — папка, относительно которой выполняется команда.
- **Абсолютный путь** начинается с `/`.
- **Относительный путь** считается от текущей директории.
- `~` — домашняя папка.
- `.` — текущая директория, `..` — родительская.
- Имена чувствительны к пробелам; заключай путь в кавычки: `cd "My Projects"`.
- Стандартный macOS-том обычно регистронезависимый, но не полагайся на это в кроссплатформенном коде.

### Команды первой необходимости

```bash
pwd                          # где я
ls                           # список файлов
ls -la                       # включая скрытые, с деталями
cd ~/Projects                # перейти в папку
cd ..                        # на уровень выше
mkdir -p ~/Projects/sandbox  # создать дерево папок
touch notes.md               # создать пустой файл/обновить timestamp
cp source.txt copy.txt       # копировать файл
cp -R dir dir-copy           # копировать директорию
mv old.txt new.txt           # переместить или переименовать
rm file.txt                  # удалить БЕЗ корзины
open .                       # открыть текущую папку в Finder
open README.md               # открыть стандартным приложением
open -a "IntelliJ IDEA" .    # открыть текущий проект в приложении
man ls                       # руководство; q для выхода
which java                   # какой executable будет запущен
type -a java                 # все совпадения/тип команды
```

`rm` не помещает файлы в Trash. Особенно опасны `rm -rf`, маски `*`, неверная текущая директория и `sudo`. Перед потенциально опасной командой выполни `pwd` и `ls`.

### Каналы, pipes и перенаправления

```bash
command > file.txt           # перезаписать stdout в файл
command >> file.txt          # дописать stdout
command 2> errors.txt        # stderr в файл
first | second               # stdout first становится stdin second
```

Примеры:

```bash
ps aux | rg java
git log --oneline | head
find . -name '*.log' 2>/dev/null
```

Кавычки важны:

- `'single quotes'` — содержимое почти буквально;
- `"double quotes"` — подставляются `$VARIABLE` и команды;
- без кавычек пробелы разделяют аргументы.

### Управление процессом

- `Ctrl+C` — отправить процессу interrupt.
- `Ctrl+Z` — приостановить текущий процесс.
- `jobs` — фоновые/приостановленные jobs текущего shell.
- `fg` — вернуть job на передний план.
- `command &` — запустить в фоне, но это не полноценный сервис-менеджмент.
- `ps aux`, `pgrep -fl java` — найти процессы.
- `kill <PID>` — попросить завершиться; `kill -9` — крайняя мера.

### Права и `sudo`

```bash
ls -l file
chmod u+x script.sh
```

`sudo` выполняет команду с повышенными правами и спрашивает пароль пользователя. При вводе пароль не отображается — даже звёздочками. Это нормально.

Правила:

- сначала понять команду без `sudo`;
- не исправлять permission error автоматическим `sudo`;
- не делать `sudo chmod -R 777 ...`;
- не менять владельца `/System`, `/Library`, `/usr`;
- Homebrew в стандартной конфигурации не требует `sudo` для обычных операций.

### `PATH` и переменные окружения

`PATH` — список директорий, где shell ищет исполняемые файлы.

```bash
echo "$PATH"
which git
env | sort
export DEMO=value
```

На Apple Silicon Homebrew добавляется примерно так:

```bash
eval "$(/opt/homebrew/bin/brew shellenv)"
```

Добавляй это один раз в рекомендуемый Homebrew файл конфигурации. Не копируй бесконечно одинаковые строки в `.zshrc`.

### Файлы настройки shell

- `~/.zprofile` — окружение login shell; часто здесь инициализируют Homebrew.
- `~/.zshrc` — настройки интерактивного zsh: aliases, prompt, completion, SDKMAN.
- `~/.zshenv` — читается почти всегда; не загружай сюда тяжёлые плагины и не меняй без необходимости.

После изменения:

```bash
source ~/.zshrc
```

Но для проверки чистого состояния лучше открыть новый Terminal.

### Удобные CLI-инструменты

```bash
brew install git ripgrep fd fzf jq yq tree htop shellcheck
```

- `rg` — быстрый поиск текста;
- `fd` — удобный поиск файлов;
- `fzf` — fuzzy picker;
- `jq`/`yq` — JSON/YAML;
- `htop` — процессы;
- `shellcheck` — анализ shell-скриптов.

Не превращай shell в хрупкий аттракцион из десятков плагинов в первый день. Сначала научись стандартному zsh.

## 13. Apple Silicon: ARM64, Rosetta и совместимость

Твой Mac использует архитектуру ARM64 (`arm64`), а многие старые Windows/Linux серверы и Intel Mac — x86_64/AMD64.

Проверка:

```bash
uname -m              # ожидается arm64
arch                  # arm64 в нативном shell
file "$(which java)"  # архитектура binary
```

### Приоритет совместимости

1. Нативная arm64-версия.
2. Universal binary с arm64 и x86_64.
3. Rosetta для старого Intel-приложения, если нет альтернативы.
4. Эмуляция x86-контейнера — только когда это действительно требуется.

Rosetta переводит Intel-код, но не превращает ARM Mac в полноценную Intel-машину и не решает все проблемы драйверов, виртуализации и контейнеров. Официальное описание: <https://developer.apple.com/documentation/apple-silicon/about-the-rosetta-translation-environment>.

### Типичные симптомы архитектурной проблемы

- `exec format error`;
- образ Docker не имеет manifest для `linux/arm64`;
- нативная библиотека JNI существует только для x86_64;
- `Bad CPU type in executable`;
- package manager скачал не ту сборку;
- проект работает в CI amd64, но падает локально;
- часть цепочки запущена под Rosetta, часть нативно, и пути расходятся.

Не начинай с принудительного `arch -x86_64`. Сначала выясни архитектуру каждого слоя: host, terminal, JDK, binary, Docker image, native dependency.

## 14. Рабочее окружение Java/Kotlin

### Рекомендуемая схема

- JetBrains Toolbox управляет версиями IntelliJ IDEA.
- SDKMAN! управляет несколькими JDK и Gradle/Maven, если это нужно.

- Homebrew ставит общие CLI, но не обязан управлять каждым JDK.

Главная идея — **один владелец каждой версии**. Не ставь один JDK через `.pkg`, второй через brew, третий через Toolbox и четвёртый вручную, если не можешь объяснить, какой выбирается и почему.

Они нужны Homebrew и проектам с native-компонентами. Полный Xcode не нужен для обычной JVM-разработки, пока проект не требует iOS/macOS SDK.

Устанавливай по официальной инструкции <https://sdkman.io/install/>. После установки:

- JVM IntelliJ IDEA;

### Настройка проекта в IDEA

Проверь:

1. **Project SDK** соответствует проекту.
2. **Gradle JVM** задан осознанно.

7. Annotation processing/Kotlin plugin соответствуют проекту.

## 15. Git и SSH

### Базовая настройка

```bash
git --version
git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global init.defaultBranch main
git config --global core.autocrlf input
git config --global core.safecrlf warn
git config --global --list --show-origin
```

Для командного проекта сначала посмотри корпоративные требования: email, commit signing, SSH/HTTPS, SSO и credential helper могут быть стандартизированы.

### SSH-ключ

Современный базовый вариант:

```bash
ssh-keygen -t ed25519 -C "you@example.com"
eval "$(ssh-agent -s)"
ssh-add --apple-use-keychain ~/.ssh/id_ed25519
pbcopy < ~/.ssh/id_ed25519.pub
```

Задай passphrase. Загрузи **публичный** ключ (`.pub`) в Git hosting. Приватный ключ никому не отправляй.

Пример `~/.ssh/config`:

```sshconfig
Host github.com
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_ed25519
  AddKeysToAgent yes
  UseKeychain yes
```

Права:

```bash
chmod 700 ~/.ssh
chmod 600 ~/.ssh/config ~/.ssh/id_ed25519
chmod 644 ~/.ssh/id_ed25519.pub
```

### Line endings и регистр

Windows часто использует CRLF, Unix/macOS — LF. `core.autocrlf input` обычно оставляет LF в репозитории, но источник правды — `.gitattributes` проекта.

Стандартный APFS часто регистронезависим. Git может не заметить простое переименование `Service.kt` → `service.kt`. Делай промежуточное имя:

```bash
git mv Service.kt temporary.kt
git mv temporary.kt service.kt
```

Это важно, потому что CI на Linux обычно регистрозависим.

## 16. Docker на Mac

### Ментальная модель

Контейнеры Docker на Mac не работают прямо на ядре macOS: Linux-контейнеры запускаются внутри лёгкой Linux VM. Поэтому:

- пути bind mount пересекают границу macOS ↔ Linux VM;
- `localhost`, сеть и filesystem performance имеют нюансы;
- лимиты CPU/RAM относятся к VM/среде Docker;
- Linux permissions и case sensitivity могут отличаться от host;
- контейнер root не равен root на macOS host, но всё равно требует осторожности.

### Установка и первый тест

Установи **Docker Desktop for Mac with Apple silicon** с официальной страницы: <https://docs.docker.com/desktop/setup/install/mac-install/>.

```bash
docker version
docker info
docker run --rm hello-world
docker run --rm alpine uname -m
docker compose version
```

В ARM64 Alpine ожидай `aarch64`.

### Настройки для 48 ГБ памяти

Не отдавай Docker «половину компьютера» заранее. Начни примерно с 6–10 ГБ RAM и 4–8 CPU для обычной backend-разработки, затем измеряй. Большие интеграционные стенды, Kafka/Elasticsearch и локальные Kubernetes-кластеры могут потребовать больше.

Включи **Resource Saver**, если Docker часто простаивает: он останавливает Linux VM и экономит ресурсы, но первый запуск контейнера после паузы займёт несколько секунд. Документация: <https://docs.docker.com/desktop/settings-and-maintenance/settings/>.

### ARM64 и образы

Предпочитай multi-arch или arm64 images:

```bash
docker buildx imagetools inspect eclipse-temurin:21-jre
docker image inspect <image> --format '{{.Architecture}}'
```

Принудительный x86:

```bash
docker run --platform linux/amd64 ...
```

используй как осознанный fallback. Эмуляция может быть медленнее и иметь несовместимости. Для публикации multi-platform image:

```bash
docker buildx build \
  --platform linux/amd64,linux/arm64 \
  -t registry.example.com/app:tag \
  --push .
```

Официальный гайд: <https://docs.docker.com/build/building/multi-platform/>.

### Производительность файлов

- Не монтируй весь `$HOME` в контейнер.
- Монтируй только каталог проекта/нужные директории.
- Для зависимостей и данных БД часто лучше named volumes.
- Не держи базы данных на bind mount в синхронизируемой iCloud-папке.
- Следи за file watchers в больших деревьях.
- `.dockerignore` должен исключать `.git`, IDE caches, build output и секреты.

### Диск Docker

```bash
docker system df
docker ps -a
docker images
docker volume ls
docker builder prune --filter until=168h  # сначала понять, что будет удалено
```

Volumes могут содержать единственную копию локальной БД. Перед удалением сделай dump, если данные важны.

### Лицензия

Docker Desktop имеет лицензионные условия для коммерческого использования. В крупной организации уточни у работодателя разрешённый вариант. Возможные альтернативы существуют, но выбирай их по требованиям команды, а не из-за моды.

## 17. AI на MacBook с 48 ГБ unified memory

### Что реально удобно

- облачные AI-клиенты для программирования, анализа и текста;
- IDE-ассистенты;
- локальный inference небольших и средних quantized-моделей;
- embeddings, reranking, transcription и локальные эксперименты;
- контейнеризированные AI-сервисы;
- прототипы с Python/MLX/PyTorch Metal при совместимых зависимостях.

Unified memory используют CPU и GPU. 48 ГБ — сильная конфигурация, но не все 48 ГБ доступны модели: часть нужна системе и приложениям. Размер файла модели не равен максимальной памяти во время inference; контекст, KV cache и runtime тоже занимают память.

### Облачные инструменты

Выбирай по политике данных:

- можно ли отправлять proprietary code;
- используется ли содержимое для обучения;
- есть ли enterprise retention/control;
- какие плагины имеют доступ к репозиторию;
- где хранятся prompts и логи;
- можно ли подключать production secrets.

Не вставляй токены, персональные данные, production dumps и закрытый код в сервис, который не одобрен компанией.

### Локальные модели через Ollama

Ollama поддерживает macOS 14+; актуальную установку смотри на <https://ollama.com/download/mac>.

Пример рабочего цикла:

```bash
ollama --version
ollama list
ollama pull <model>
ollama run <model>
ollama ps
ollama rm <model>
```

Начни с небольшой модели, замерь скорость и качество, затем увеличивай размер. Следи за:

- свободным диском;
- memory pressure в Activity Monitor;
- размером контекста;
- температурой и энергопотреблением;
- тем, слушает ли локальный API только loopback, если сервис содержит чувствительные данные.

Локальная модель повышает приватность относительно облака, но не отменяет безопасность: prompts, история и model cache остаются на диске; вредоносная модель/инструмент может быть риском; подключение model tool к shell или файлам требует минимальных прав.

### Docker Model Runner и контейнеры

Если AI-проект уже строится вокруг Docker, изучи Model Runner: <https://docs.docker.com/ai/model-runner/>. Не держи одновременно несколько тяжёлых runtimes и одну и ту же модель в памяти без причины.

### Практичная раскладка памяти

При тяжёлом локальном inference:

- закрой лишние вкладки браузера;
- останови ненужные Compose-стенды;
- не выделяй IDEA гигантский heap;
- наблюдай **Activity Monitor → Memory → Memory Pressure**, а не только число «used»;
- избегай постоянного сильного swap при длительной нагрузке.

## 20. Автоматизация: от Shortcuts до shell

Выбирай самый простой слой, который надёжно решает задачу:

1. встроенная функция приложения;
2. keyboard shortcut;
3. Shortcuts;
4. shell script;
5. `launchd` для фоновой задачи;
6. специализированный инструмент автоматизации.

### Shortcuts

Хорошие первые сценарии:

- включить рабочий Focus и открыть IDEA, Terminal, браузер;
- подготовить режим созвона;
- конвертировать/переименовать выбранные файлы;
- создать заметку по шаблону;
- скопировать текущую дату/UUID;
- выполнить безопасный shell script на выбранных файлах.

Проверяй входные данные и область доступа. Shortcut из интернета — это программа; просмотри все действия перед запуском.

### Shell script

Минимальный безопасный шаблон:

```bash
#!/usr/bin/env bash
set -euo pipefail

main() {
  printf 'Hello, %s\n' "${USER}"
}

main "$@"
```

```bash
chmod u+x script.sh
./script.sh
shellcheck script.sh
```

`set -euo pipefail` полезен, но не магическая гарантия; понимай обработку ошибок и quoting.

### launchd

macOS использует `launchd`, а не systemd. Пользовательские агенты обычно описываются plist-файлами в `~/Library/LaunchAgents`. Не копируй старые команды `launchctl` без проверки текущей документации. Для простой периодической задачи часто лучше корпоративный scheduler, CI или Shortcuts automation.

## 21. Диагностика: что делать, когда «что-то не работает»

### Универсальный порядок

1. Зафиксируй симптом и точный текст ошибки.
2. Определи область: одно приложение, пользователь, сеть, диск, вся система.
3. Проверь недавние изменения: update, VPN, plugin, permissions, новый monitor/dock.
4. Сохрани важные данные.
5. Перезапусти конкретное приложение.
6. Проверь Activity Monitor и свободное место.
7. Перезагрузи Mac.
8. Обнови приложение и macOS до стабильных версий.
9. Отключи по одному сторонние login items, VPN, filters, IDE plugins.
10. Проверь в другом user account или Safe Mode, если проблема системная.
11. Только затем переустанавливай или делай более глубокие изменения.

### Зависло приложение

- `⌘⌥Esc` → выбрать приложение → Force Quit.
- Или Activity Monitor → процесс → Quit, затем Force Quit.
- В Terminal: `pgrep -fl AppName`, потом обычный `kill PID`.
- `kill -9` оставь на случай, когда нормальное завершение не работает.

### Mac тормозит

Открой Activity Monitor:

- **CPU:** кто постоянно загружает ядра;
- **Memory:** смотри Memory Pressure, swap и процессы;
- **Energy:** кто разряжает батарею;
- **Disk:** интенсивная запись/чтение;
- **Network:** неожиданный трафик.

Возможные причины у разработчика: indexing IDEA, Gradle daemon, Docker VM, browser tabs, local LLM, Spotlight indexing после миграции, cloud sync, antivirus/EDR.

Не делай вывод по одному мгновенному пику. Смотри устойчивую нагрузку и воспроизводимость.

### Заканчивается место

1. System Settings → General → Storage.
2. `df -h /`.
3. Downloads, Trash, большие видео/образы.
4. `docker system df`.
5. AI models.
6. Gradle/Maven caches и старые JDK — удалять средствами соответствующего менеджера.
7. Time Machine local snapshots обычно управляются системой; не начинай с ручного удаления.

### Сеть

```bash
ifconfig
networkQuality
ping -c 4 1.1.1.1
ping -c 4 example.com
dig example.com
curl -I https://example.com
lsof -nP -iTCP -sTCP:LISTEN
```

Интерпретация:

- IP пингуется, домен нет — вероятен DNS;
- браузер работает, CLI нет — proxy/certificate/environment;
- только рабочие ресурсы не работают — VPN/SSO/routes;
- порт занят — `lsof` покажет процесс;
- после VPN сломалось всё — отключи VPN/filter и проверь снова, если политика позволяет.

### «Приложение не видит файл/камеру/экран»

Проверь Privacy & Security. После изменения разрешения приложение иногда нужно полностью завершить через `⌘Q` и открыть заново.

### «Команда не найдена»

```bash
echo "$PATH"
type -a commandname
which commandname
brew list | rg commandname
```

Открой новый shell. Проверь, не установил ли пакет binary с другим именем, и не запущен ли Terminal под Rosetta.

### Recovery и Safe Mode на Apple Silicon

- Выключи Mac.
- Удерживай кнопку питания, пока не появятся startup options.
- Оттуда доступны Options/Recovery и выбор диска.
- Для Safe Mode выбери volume, удерживай Shift и выбери продолжение в Safe Mode.

Recovery позволяет переустановить macOS, запустить Disk Utility и изменить некоторые параметры безопасности. Переустановка macOS обычно не является первым шагом диагностики.

## 22. Внешние мониторы, docks и периферия

- Проверяй возможности конкретного чипа/модели Mac и dock: количество дисплеев, разрешение, частоту, HDR, DisplayPort/HDMI версию.
- USB‑C — форма разъёма, а не гарантия скорости, видео или мощности.
- Thunderbolt-кабель и обычный charging USB‑C cable могут выглядеть одинаково, но вести себя по-разному.
- Для стабильной зарядки используй адаптер достаточной мощности и качественный кабель.
- Если монитор странно масштабируется, попробуй системные варианты resolution/scaling; «больше пространства» требует больше от GPU, но обычно нормально.
- Закрытая крышка (clamshell) обычно требует питания и внешних клавиатуры/мыши; следи, чтобы вентиляция не перекрывалась.
- Для веб-камер, микрофонов и capture cards проверь Privacy permissions.
- Если dock глючит: исключи цепочку — другой порт, другой кабель, прямое подключение, питание dock, обновление firmware.

Не покупай dock только по количеству нарисованных портов. Сначала выпиши: число мониторов, 4K/5K и Hz, Ethernet speed, SD, power delivery, Thunderbolt devices.

## 23. Рабочая организация и продуктивность

### Простая структура файлов

```text
~/Projects/          git repositories
~/Documents/Work/    документы, не являющиеся репозиториями
~/Documents/Personal/
~/Downloads/         временный входящий поток, регулярно разбирать
~/Archive/           завершённое, если не в облачном архиве
```

Не держи активные Git-репозитории в iCloud-synced Desktop/Documents, если команда инструментов создаёт много временных файлов. Это может давать конфликты, лишнюю синхронизацию и расход диска.

### Focus и уведомления

Создай режимы:

- **Deep Work:** только календарь и критичные контакты;
- **Meetings:** разрешён meeting client, скрыты сообщения на экране;
- **Personal:** рабочие приложения приглушены.

Отключи уведомления у приложений, которые не требуют реакции. Badge — тоже уведомление. Опытный пользователь не обязан постоянно видеть каждое событие.

### Меню-бар и Login Items

System Settings → General → Login Items & Extensions:

- оставь только то, что должно стартовать всегда;
- различай «Open at Login» и фоновые helper items;
- после удаления приложения проверь, не остался ли helper;
- неизвестный элемент сначала идентифицируй, а не удаляй наугад.

### Быстрый рабочий ритуал

1. Включить Work Focus.
2. Открыть текущий проект и терминал.
3. Проверить календарь один раз.
4. Запустить нужный Compose stack, а не весь зоопарк.
5. В конце: commit/push по правилам, остановить ненужные контейнеры, записать next action.

## 24. Антипаттерны новичка

1. **Закрывать окна красной кнопкой и удивляться, что приложение живо.** Используй `⌘Q`, когда хочешь завершить.
2. **Открывать всё через Dock.** Освой `⌘Space`.
3. **Перетаскивать каждое окно мышью.** Используй Spaces, Mission Control и tiling.
4. **Устанавливать десятки «must-have apps» в первый день.** Сначала обнаружь потребность.
5. **Запускать любой fix с `sudo`.** Permission error — симптом, а не инструкция.
6. **Ставить Intel-версию при наличии arm64.** Проверяй архитектуру.
7. **Считать iCloud бэкапом.** Добавь Time Machine и восстановительный тест.
8. **Чистить кэши ради красивого числа.** Кэш ускоряет работу и обычно восстановится.
9. **Отдавать Docker/IDE всю память.** Настраивай по измерениям.
10. **Хранить единственную локальную БД в Docker volume.** Делай dump важных данных.
11. **Отключать системную безопасность ради старой утилиты.** Ищи современную альтернативу.
12. **Смешивать JDK-менеджеры.** Определи один источник версии.
13. **Пытаться превратить macOS в Windows пиксель-в-пиксель.** Сохрани полезные привычки, но освой модель Mac.
14. **Обновляться на beta OS на основной рабочей машине.** Для стабильной работы используй production-релизы и сначала делай бэкап.

## Вместо заключения

Опытный пользователь macOS — не тот, кто установил больше твикеров и помнит все скрытые команды. Это тот, кто:

- быстро выражает намерение через клавиатуру, поиск и автоматизацию;
- понимает границы приложения, процесса, пользователя, системы, VM и контейнера;
- сохраняет нативную архитектуру и воспроизводимость окружения;
- умеет измерить проблему до «лечения»;
- выдаёт минимальные разрешения;
- имеет рабочий, проверенный план восстановления.

Освой сначала штатную логику Mac. Затем меняй только то, что решает твою конкретную повторяющуюся проблему. Так система останется быстрой, предсказуемой и действительно твоей.
