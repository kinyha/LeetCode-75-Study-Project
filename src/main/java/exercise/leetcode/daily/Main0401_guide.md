# 2751. Robot Collisions
**Difficulty:** Hard  
**Topics:** Array, Stack, Sorting, Simulation

## Условие задачи (краткое)

На прямой линии находятся роботы, каждый имеет позицию, здоровье и направление движения (L или R). Все двигаются одновременно с одинаковой скоростью. При столкновении робот с меньшим здоровьем уничтожается, а у выжившего здоровье уменьшается на 1. Если здоровье одинаковое — оба уничтожаются. Нужно вернуть здоровье выживших роботов **в исходном порядке**.

## Необходимая теория

### Stack (Стек)

- **Что это:** структура данных LIFO (Last In, First Out) — последний вошёл, первый вышел
- **Основные операции:** `push` (добавить на вершину), `pop` (удалить с вершины), `peek` (посмотреть вершину) — все за O(1)
- **Когда полезен:** когда нужно обрабатывать "встречные" элементы и разрешать конфликты между ними
- **Аналогия из жизни:** стопка тарелок — ты всегда берёшь верхнюю. Или скобочные последовательности — открывающая скобка кладётся в стек, при встрече закрывающей — сопоставляется с верхней
- **Классический паттерн:** задача "Asteroid Collision" (LC 735) — почти идентичная идея. Астероиды летят навстречу, сталкиваются, выживает более крупный

### Sorting с сохранением индексов

- **Что это:** сортировка элементов, при которой ты помнишь, откуда каждый элемент пришёл
- **Зачем:** когда данные нужно обработать в одном порядке (например, по позиции), но результат вернуть в другом (в исходном порядке)
- **Как реализовать:** создать массив индексов [0, 1, 2, ..., n-1] и отсортировать его по значениям из другого массива. Или создать объекты/кортежи (index, value) и сортировать их
- **Сложность:** O(n log n) для сортировки

### Simulation (Симуляция)

- **Что это:** пошаговое моделирование процесса, описанного в задаче
- **Ключевая идея:** не нужно буквально симулировать движение по шагам. Вместо этого подумай: **в каком порядке происходят столкновения** и можно ли определить это заранее
- **Важно:** скорость одинаковая, значит столкнуться могут только роботы, которые движутся навстречу друг другу (R...L) и между ними нет других роботов

## Как подойти к решению

### Наводящие вопросы (прочитай перед тем как начать)

1. Если все роботы движутся в одну сторону, может ли произойти столкновение? А если чередуются R и L?
2. Какие именно пары роботов могут столкнуться? Может ли R-робот столкнуться с L-роботом, который стоит **левее** него?
3. Если отсортировать роботов по позиции и идти слева направо — что означает встреча R-робота? А L-робота?
4. Какую задачу эта напоминает? Подумай про скобки или астероиды.
5. Как не потерять исходный порядок роботов при том, что обработка идёт в порядке позиций?

### На что обратить внимание

- **Позиции не отсортированы** — это прямо сказано в условии. Нельзя просто идти по массиву
- **Столкновения каскадны** — робот может пережить одно столкновение (потеряв 1 HP) и тут же столкнуться с другим
- **Результат в исходном порядке** — это не порядок позиций, а порядок в котором роботы были даны на вход
- Столкновение происходит только между R (идущим вправо) и L (идущим влево), причём R должен быть **левее** L
- Один L-робот может уничтожить **несколько** R-роботов подряд, если у него достаточно здоровья

### Edge Cases

- Все роботы движутся в одну сторону (нет столкновений)
- Все роботы имеют одинаковое здоровье и чередуются R-L (все уничтожаются)
- Один очень "толстый" робот (с огромным здоровьем) уничтожает всех встречных
- Только два робота
- n = 1 (один робот, столкновений нет)

### Целевая сложность

- **Time:** O(n log n)
- **Space:** O(n)

## Подсказки (открывай по одной, только если застрял)

### Подсказка 1 (легкая — направление)

> Эта задача решается с помощью **стека**. Вспомни задачу "Asteroid Collision" — здесь очень похожая механика. Подумай, какие элементы класть в стек и когда их оттуда доставать.

### Подсказка 2 (средняя — структура)

> Отсортируй роботов по позиции (слева направо). Иди по отсортированному порядку: R-роботов клади в стек. Когда встречаешь L-робота — начинай "столкновения" с вершиной стека. Не забудь сохранить исходные индексы, чтобы в конце восстановить правильный порядок.

### Подсказка 3 (сильная — ключевая идея)

> При встрече L-робота с вершиной стека (R-роботом) сравни их здоровье:
> - Если здоровье R **меньше** — убери R из стека, уменьши здоровье L на 1, продолжай проверять стек
> - Если здоровье **равно** — убери R из стека, L тоже уничтожен, переходи к следующему роботу
> - Если здоровье R **больше** — L уничтожен, уменьши здоровье R на 1, R остаётся в стеке
> 
> Этот цикл для одного L-робота может убрать несколько R-роботов из стека.

### Подсказка 4 (почти решение — псевдокод)

> 1. Создать массив индексов [0..n-1], отсортировать по позициям
> 2. Создать стек (хранит индексы роботов, идущих вправо)
> 3. Для каждого робота в порядке позиций:
>    - Если направление = R → положить индекс в стек
>    - Если направление = L → пока стек не пуст и вершина = R-робот:
>      - Сравнить здоровье, обработать столкновение (см. подсказку 3)
>    - Если L-робот выжил (стек опустел или вершина тоже L) → запомнить его как выжившего
> 4. Все роботы оставшиеся в стеке тоже выжили
> 5. Собрать выживших, отсортировать по исходному индексу, вернуть их здоровье
