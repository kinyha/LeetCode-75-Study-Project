# Задание 4.1: циклические зависимости в Spring

## Дано

В пакете лежит намеренно проблемный пример Spring-кода:

- `OrderService` зависит от `PaymentService`.
- `PaymentService` зависит обратно от `OrderService`.
- обе зависимости внедряются через field injection.

## Задача

Отрефакторить код так, чтобы сервисы больше не образовывали циклическую зависимость.

Простое бизнес-поведение должно сохраниться:

- размещение заказа все еще должно запускать списание оплаты;
- ошибка оплаты все еще должна приводить к отмене заказа.

Не считай `@Lazy` или включение circular references финальным решением.
